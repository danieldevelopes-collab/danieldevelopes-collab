## How these repositories are built

Every project here aims for the same standard:

1. **Deliberate design** — what it does, what the architecture should be, what the user experience should feel like, what trade-offs are acceptable.
2. **End-to-end ownership** — the code, the tests, the docs, the release.
3. **Honesty is non-negotiable** — every README clearly states what works vs. what is platform-dependent, what was verified vs. assumed.

If something looks polished, it was directed to be. If something is missing, it was honestly out of scope.
